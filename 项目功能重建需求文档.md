# 天枢·AI临床决策管理系统 — 功能重建需求文档

> 本文档不含任何技术实现细节，仅描述系统"要实现什么功能"，供你使用其他智能体从头重建时参考。

---

## 一、系统总体定位

这是一套面向医院/诊所的综合性临床业务管理系统，深度融合了中医特色诊疗和 AI 辅助决策能力。系统覆盖患者从入院到出院的全生命周期管理，包含药品管理、治疗排班、随访宣教、请假审批等日常业务，同时提供异质性疗效预测（HTE）、智能预问诊、风险哨兵、OCR 处方识别、二维码安全传输等智能化功能。

---

## 二、用户角色与权限体系

系统需要支持以下 4 种用户角色，不同角色拥有不同的功能访问权限：

### 1. 管理员（admin）
- 全部功能访问权限
- 用户账号管理（创建/编辑/删除用户，配置角色和功能权限）
- 系统设置管理
- 操作审计日志查看

### 2. 医生（doctor）
- 患者管理（查看、编辑、创建患者）
- 治疗方案制定与管理
- 医嘱开具与药品分配
- 随访计划创建与执行
- 请假审批
- AI 决策分析查看
- 排班与预约管理

### 3. 护士（nurse）
- 患者基本信息查看
- 药品发放与领取确认
- 随访数据录入
- 健康宣教内容查看

### 4. 患者（patient）
- 个人信息查看
- 自助注册（需邀请码）
- 预问诊问卷填写
- 健康信息采集
- 请假/外出申请
- 健康反馈提交
- 穴位知识库查询

## 三、功能模块详述

---

### 模块 1：用户认证与注册

#### 1.1 用户登录
- 用户名 + 密码登录
- 支持双因素认证（TOTP 动态码）
- 登录成功后返回 JWT 令牌
- 区分管理员/医生/护士/患者不同入口

#### 1.2 患者自助注册
- 需输入 8 位邀请码进行验证
- 邀请码验证通过后方可填写注册信息
- 注册信息包含：用户名、密码、姓名、所属医院、所属科室
- 注册完成后自动成为"患者"角色

#### 1.3 用户管理（管理员功能）
- 查看所有用户列表
- 按角色筛选用户
- 创建新用户：设置用户名、密码、姓名、角色、医院、科室
- 编辑用户：修改信息、分配功能权限（12 项细粒度权限复选框）
- 删除用户
- 医院/科室输入支持自动补全和历史记录

---

### 模块 2：患者管理

#### 2.1 患者列表
- 以表格形式展示所有患者
- 展示字段：姓名、性别、年龄、床位号、科室、诊断、匿名 ID、创建时间、风险状态、风险警告
- 按患者姓名或匿名 ID 搜索
- 高危患者红色标记
- 响应式设计：大屏显示完整操作按钮，小屏仅显示图标

#### 2.2 患者信息管理
- 创建患者：录入姓名、性别、年龄、联系电话、地址、身份证号、诊断、入院日期、床位号、科室
- 编辑患者信息
- 删除患者
- 患者信息自动生成匿名化 ID
- 敏感信息（姓名、电话、身份证号）支持加密存储与脱敏展示

#### 2.3 患者分类管理（住院/门诊/慈善）
- 将患者分为住院、门诊、慈善三类进行管理
- 每类患者独立列表视图，附带数量统计
- 支持患者类型的转换
- 患者出院操作
- 患者归档操作
- 各类患者的统计概览卡片

#### 2.4 患者隐私与数据安全
- 患者信息展示时自动脱敏（姓名掩码、电话掩码、身份证号掩码）
- 支持授权用户输入解密密钥查看真实信息
- 解密操作记录审计日志

#### 2.5 二维码功能
- 为每个患者生成信息二维码（支持分块传输长内容）
- 支持上传二维码图片进行解码，获取患者信息

---

### 模块 3：预问诊与健康信息采集

#### 3.1 智能预问诊
- 患者就诊前填写预问诊表单
- 采集内容：主诉、症状（多选）、病史、家族史、生活习惯
- 采集患者隐性偏好：疼痛耐受度（滑动条 1-10）、治疗速度偏好（快速/循序渐进/稳健）、沟通需求程度、副作用耐受度、价格敏感度
- 提交后自动进行 NLP 结构化处理、风险评估、隐式偏好分析
- 自动创建或关联患者记录

#### 3.2 完整健康信息采集（三步向导）
- 步骤一 - 基本信息：姓名、性别、年龄、联系电话、职业、文化程度、婚姻状况、吸烟史、饮酒史
- 步骤二 - 病史信息：主诉、现病史、既往史、过敏史、用药史、家族史、运动习惯、睡眠习惯、饮食习惯、疼痛耐受度、治疗速度偏好、复诊偏好（线上/线下/均可）
- 步骤三 - 确认提交：展示已填信息摘要，确认后提交保存
- BMI 自动计算

#### 3.3 患者健康档案
- 每个患者拥有一份完整的健康档案
- 包含：用药历史、就诊记录、检验检查结果、诊断记录
- 医生可查看和更新患者健康档案

#### 3.4 隐私偏好与智能配置
- 患者可设置隐私偏好：匿名化展示开关、数据共享许可等级、通知偏好、研究用途许可
- 患者可配置智能辅助功能：AI 辅助诊断开关、个性化推荐开关、用药提醒开关、健康预警开关、智能复诊提醒开关、数据驱动治疗优化开关

---

### 模块 4：药品管理

#### 4.1 药品字典管理
- 药品基础信息管理：药品名称、规格、剂型、单位、默认用量、用法、价格、生产厂商、备注
- 支持药品的增删改查
- 药品分类：精神类/西药/中成药
- 精神类药品标识

#### 4.2 药品库存管理
- 记录每种药品的当前库存总量
- 设置最小库存预警值
- 低库存药品自动预警提示
- 库存状态用颜色编码：<=1 天用量红色紧急、<=3 天橙色预警、>3 天绿色正常

#### 4.3 药品发放全流程管理
- 流程：患者医嘱 → 领药医嘱 → 药品领取登记 → 药品发放确认
- 每一步骤独立管理，附带数量统计
- 支持按状态筛选、按患者 ID 搜索
- 低剂量预警提示

#### 4.4 发药记录
- 记录每次发药：患者、药品名称、规格、每日剂量、领取数量、剩余药量、发药日期、预计下次领药日期、发药人
- 自动计算剩余药量
- 下次领药日期自动推算
- 生成防篡改签名（SHA-256）

#### 4.5 科室级用药管理（医生/护士分工）
- 医生角色：查看所有医嘱、新建医嘱、编辑医嘱、停止医嘱
- 护士角色：查看待发药列表、记录发药操作、标记未发药（原因可选：患者拒绝/外出/检查/缺货/其他）
- 统计卡片：活跃医嘱数、待发药数、已发药数、未发药数

---

### 模块 5：治疗管理

#### 5.1 治疗项目词典
- 管理各种治疗项目的基础信息
- 治疗项目分类（11 种）：针灸、推拿、艾灸、拔罐、刮痧、理疗、心理咨询、心理治疗、药物治疗、手术治疗、康复训练
- 每个项目包含：名称、代码、分类、标准时长、标准频次、价格、描述、所需设备、启用/禁用状态
- 支持自定义标记

#### 5.2 治疗方案价格管理
- 治疗项目的定价管理
- 支持增删改查

#### 5.3 中医特色治疗管理
- 管理中医特色治疗项目：艾灸、拔罐、刮痧、耳穴、针灸、推拿
- 每个治疗包含：名称、分类、描述、适应症、禁忌症、建议时长、参考价格、自定义标记

#### 5.4 穴位管理（中医针灸知识库）
- 管理穴位信息，包含丰富的知识属性
- 穴位基本信息：名称、拼音、英文名、代码、归经、分类
- 穴位定位：位置描述、简易定位法、解剖结构
- 穴位主治：主治功能、现代临床应用、中医辨证主治
- 穴位操作：刺法、灸法、按摩手法
- 配伍关系：常用配伍穴位、经典处方举例
- 安全信息：注意事项、禁忌、警告
- 学习资源：穴位图解、教学视频、参考文献、临床案例、研究论文、现代研究进展
- 穴位特性：功效列表、五行/阴阳属性、相关经脉、相关脏腑、相关证型
- 用户互动：浏览次数统计、用户评分

#### 5.5 针刺手法管理
- 管理各种针刺手法
- 包含：手法名称、描述、作用功效、操作方法、留针时间

#### 5.6 治疗计划管理
- 制定综合治疗方案计划
- 包含：针灸处方（选穴+手法）、中医治疗（项目+时长）、药物治疗（药品+剂量+用法）、其他治疗
- 支持设置：疗程次数、治疗频率、开始日期、结束日期
- 治疗计划状态流转：草稿 → 执行中 → 已完成 / 已停止
- 治疗计划关联到具体患者和病历
- 疗效评估与评分

#### 5.7 三级治疗管理系统
- 治疗类型 → 治疗项目 → 治疗计划三级管理体系
- 每种治疗类型可包含多个治疗项目
- 每个治疗计划可包含多个治疗项目
- 统计概览：总数、执行中、已完成、草稿数量
- 治疗类型分布图

---

### 模块 6：排班与预约管理

#### 6.1 医生排班管理
- 按星期设置医生的可用时间段
- 排班包含：星期几、开始时间、结束时间、是否可用
- 支持单个班次设置和批量生成班次（按时间间隔自动生成连续班次）

#### 6.2 治疗预约管理
- 患者预约治疗项目
- 预约时选择：医生、治疗项目、预约日期、时间段
- 自动检查医生排班和时段冲突
- 预约状态：已确认、已签到、已取消、已完成、未到诊
- 取消预约规则：提前一定时间可免费取消，超时自动计费

#### 6.3 排班总览看板
- 统计卡片：预约总量、已确认数、可预约数、资源利用率（含进度条）
- 治疗类型分布图
- 班次列表（颜色编码，显示可预约状态）
- 日历视图（Badge 显示每日预约数）
- 支持导出 Excel / PDF

---

### 模块 7：随访管理

#### 7.1 随访计划管理
- 医生为患者创建随访计划
- 随访类型：线上随访/门诊随访/电话随访
- 设置计划随访日期
- 随访计划关联到具体的病历记录
- 随访状态：待执行/已完成/已错过

#### 7.2 随访执行
- 记录随访结果：当前症状、用药依从性、不良反应、血压、心率
- 医生评估与治疗调整方案
- 确定下次随访日期
- 患者评分与意见反馈

#### 7.3 随访看板
- 今日待随访列表
- 各状态随访数量统计
- 按患者/状态筛选随访记录

---

### 模块 8：健康宣教管理

#### 8.1 宣教文章管理
- 创建和管理健康教育文章
- 文章分类：饮食、运动、心理、作息、疾病知识
- 文章标签与适用疾病标注
- 文章状态：草稿/待审核/已发布/已拒绝
- 审核流程
- 推送控制：是否可推送、目标患者群体

#### 8.2 文章展示
- 文章列表浏览与搜索
- 文章详情查看（含浏览次数统计）

#### 8.3 患者分组
- 创建患者分组（用于定向推送宣教内容）
- 分组管理

---

### 模块 9：请假与外出管理

#### 9.1 患者请假/外出申请
- 患者提交请假或外出申请
- 申请类型：请假/外出
- 填写：开始时间、结束时间、原因、目的地（外出）、紧急联系电话
- 外出需记录预计返回日期，支持返回确认
- 患者可取消待审批的申请
- 申请状态：待审批/已批准/已拒绝

#### 9.2 医生审批
- 查看待审批申请列表
- 审批操作：通过/拒绝，附带审批意见
- 各状态申请数量统计

---

### 模块 10：个人中心

#### 10.1 用户信息展示
- 显示当前登录用户的基本信息：头像、用户名、角色、科室、医院
- 账号安全设置

#### 10.2 医生档案
- 医生专业信息管理：科室、医院、专业特长
- AI 诊疗风格特征配置
- 实践画像管理
- 历史疗效追踪：总病例数、成功病例数、成功率、各治疗方式统计
- 医患匹配建议：基于专科关键词和风格评分推荐匹配患者

---

### 模块 11：AI 智能决策

#### 11.1 异质性治疗效果预测（HTE）
- 基于患者特征（病情、病史、偏好等）和医生特征（风格、专长等）
- 对不同的候选治疗方案进行疗效预测
- 预测结果包含：成功率、置信区间、预期疗程、预期费用
- 反事实对比：展示不同方案之间的预测结果对比
- 医生-患者匹配协同度评分
- 治疗轨迹预测：逐周改善曲线
- 概率云分布数据

#### 11.2 风险哨兵评估
- 对患者数据进行多维度风险评估
- 返回：风险类型、风险等级（低/中/高/极高）、具体风险描述、处理建议、整体风险评分（0-1）
- 自动标记高风险患者

#### 11.3 智能预问诊分析
- 将患者自由文本主诉进行结构化处理
- 自动提取症状、病史关键词
- 结合患者信息进行隐式偏好分析
- 生成结构化病历数据

#### 11.4 OCR 处方识别
- 上传处方图片（Base64 格式）
- 识别图片中的文字内容
- 将识别结果进行结构化处理
- 医学术语标准化映射（如药品商品名→通用名）

---

### 模块 12：二维码安全传输

#### 12.1 二维码生成
- 将患者信息/文本内容生成二维码图片
- 超长内容自动分块，生成多个二维码
- 支持 Base64 图片格式输出

#### 12.2 二维码解码
- 上传二维码图片进行解码
- 多块内容自动排序拼接
- 还原原始内容

---

### 模块 13：日志审计

#### 13.1 操作审计日志
- 记录所有关键操作的日志：操作类型、操作人、会话 ID、IP 地址
- 资源类型和资源 ID 记录
- 操作详情 JSON 存储
- 操作时间戳
- 操作成功/失败状态，失败原因记录

#### 13.2 日志查看器
- 日志表格展示
- 按用户 ID/操作类型/模块/时间范围筛选
- 成功/失败统计卡片
- 查看日志详情（含完整操作内容 JSON、IP、浏览器信息、错误信息）

## 四、业务数据实体清单（纯业务描述）

以下列出系统中管理的所有业务数据实体，不包括任何技术字段：

### 核心实体
1. **用户**：用户名、邮箱、密码(加密)、姓名、角色、是否激活、双因素密钥、科室、医院
2. **患者**：匿名 ID、姓名(可加密)、性别、年龄、电话(可加密)、地址、身份证号(可加密)、诊断、入院日期、床位号、科室、主管医生、备注、患者特征、患者画像、隐式偏好、风险评估
3. **医生**：关联用户、科室、医院、专业特长、AI 风格特征、实践画像、历史疗效
4. **病历**：关联患者、诊断、症状、病史、治疗方案、医生风格特征、HTE 预测结果、反事实对比、随访安排、风险标记、患者反馈、治疗结果

### 业务实体
5. **药品**：名称、通用名、分类(精神类/西药/中成药)、规格、单位、总库存、最小库存预警值、生产厂家、描述
6. **发药记录**：关联患者、关联药品、药品名称、规格、每日剂量、领取数量、剩余药量、发药日期、下次领药日期、发药人、防篡改签名
7. **治疗项目**：名称、时长、价格、分类(针灸/推拿/理疗/心理等)、描述、提前取消时限
8. **医生排班**：关联医生、星期几、开始时间、结束时间、是否可用
9. **预约**：关联患者、关联医生、关联治疗项目、预约日期、时间段、状态、取消信息、是否收费
10. **治疗方案**：关联患者、关联病历、方案名称、辨证诊断、治则治法、穴位处方、中医治疗、药物治疗、疗程、频率、起止日期、状态、疗效评估
11. **随访记录**：关联患者、关联病历、关联医生、计划日期、随访类型、状态、当前症状、用药依从性、不良反应、生命体征、医生评估、治疗调整、下次随访日期、患者评分
12. **请假申请**：关联患者、起止时间、类型(请假/外出)、原因、目的地、状态、审批信息、返回确认、防篡改签名
13. **健康档案**：关联患者(一对一)、用药历史、就诊记录、检验检查结果、诊断记录
14. **宣教文章**：标题、内容、分类、标签、适用疾病、推送控制、创建人
15. **健康任务**：关联患者、标题、描述、频率(每日/每周/每月)、内容、起止日期、提醒时间
16. **健康反馈**：关联患者、关联任务、反馈日期、表单数据
17. **穴位**：名称、拼音、英文名、代码、归经、分类、位置、主治、刺法、灸法、配伍、禁忌、功效、属性等
18. **针刺手法**：名称、描述、功效、操作方法、留针时间
19. **中医治疗项目**：名称、分类(艾灸/拔罐/刮痧/耳穴/针灸/推拿)、描述、适应症、禁忌症、时长、价格
20. **审计日志**：操作类型、操作人、会话、IP、资源类型、资源 ID、操作详情、时间、成功/失败状态

## 五、系统核心业务流

### 流程一：患者就诊全流程
```
患者自助注册(邀请码) → 预问诊(填问卷) → 健康信息采集 → 
医生查看患者信息 + AI 决策分析(HTE预测) → 制定治疗方案 → 
执行治疗(预约排班) → 发药管理 → 随访跟踪 → 健康宣教
```

### 流程二：药品管理流程
```
药品字典维护 → 医生开医嘱 → 生成领药医嘱 → 
患者/护士领取登记 → 药品发放确认 → 库存自动扣减 → 
低库存预警
```

### 流程三：排班预约流程
```
医生设置排班 → 患者预约时间段 → 系统自动排重 → 
预约确认 → 执行治疗 → 完成/取消/未到诊标记
```

### 流程四：请假外出流程
```
患者提交申请 → 医生审批 → 患者外出/请假 → 
患者返回确认 → 记录归档
```

### 流程五：AI 决策辅助流程
```
患者数据采集 → 风险哨兵评估(自动) → 
医生选择候选治疗方案 → HTE 模型预测各方案效果 → 
反事实对比展示 → 医生综合决策 → 制定治疗方案
```

## 六、系统特性与约束要求

### 数据安全要求
- 患者敏感信息（姓名、电话、身份证号）需加密存储
- 数据展示需自动脱敏掩码
- 查看真实信息需授权并提供解密密钥
- 解密操作需记录审计日志
- 患者数据支持生成匿名化 ID

### 数据防篡改要求
- 药品发药记录需生成防篡改签名
- 请假申请记录需生成防篡改签名
- 审计日志不可删除或修改

### 响应式与体验要求
- 页面需适配不同屏幕尺寸（桌面/平板/手机）
- 大屏显示完整操作按钮，小屏收起为图标
- 状态信息使用颜色编码标识

### 角色权限要求
- 不同角色看到不同的功能菜单和操作按钮
- 医生和护士在药品管理中有明确分工
- 患者只能查看和操作自己的数据

## 七、AI 引擎能力清单

1. **HTE 异质性疗效预测**：基于因果森林算法，预测不同治疗方案对特定患者的疗效差异
2. **语义相似度计算**：医学术语之间的语义相似度矩阵计算
3. **最近邻语义匹配**：给定查询词，从候选词列表中找出语义最接近的匹配
4. **术语标准化映射**：药品商品名到通用名的映射（内置常见药品映射库）
5. **风险评估**：多维度患者风险因素评估
6. **NLP 文本结构化**：将自由文本主诉/对话转为结构化病历格式
7. **隐式偏好分析**：从患者填写信息中分析治疗偏好倾向
